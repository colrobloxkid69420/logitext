COMPONENTS SUMMARY
==================

LAYOUT COMPONENTS (3)
  Header.svelte          → Responsive header with logo + user menu
  Sidebar.svelte         → Collapsible sidebar with navigation
  Footer.svelte          → Footer with links and copyright

UI COMPONENTS (6)
  Button.svelte          → Primary, secondary, outline, danger variants
  Card.svelte            → Reusable card container with title
  Input.svelte           → Form input with label and error support
  Modal.svelte           → Modal dialog with backdrop
  Tabs.svelte            → Tab navigation component
  Alert.svelte           → Status alerts (info, success, warning, error)

PAGE COMPONENTS (5)
  Dashboard.svelte       → Overview with cards and statistics
  Editor.svelte          → Main editor with 4 sub-panels
                          - Proof canvas
                          - Proof steps
                          - Available rules
                          - Console output
  Projects.svelte        → Project/proof management with table
  Settings.svelte        → User account and preferences
  Help.svelte            → Documentation and FAQs

ROUTING STRUCTURE
================

Hash-based routing implemented in Router.svelte

Routes:
  /                      → Dashboard (default)
  #dashboard            → Dashboard page
  #editor               → Proof Editor
  #projects             → Projects & Proofs
  #settings             → Settings page
  #help                 → Help & Documentation

STYLING SYSTEM
==============

Tailwind CSS v4 with custom configuration

Color Palette:
  Primary:   #2563eb (Blue)
  Secondary: #7c3aed (Purple)
  Accent:    #06b6d4 (Cyan)
  Success:   #10b981 (Green)
  Warning:   #f59e0b (Amber)
  Error:     #ef4444 (Red)

Neutral Scale: 50-900 (Light to Dark)

Spacing: 8px base unit (xs, sm, md, lg, xl, 2xl)
Typography: 4 heading levels + body text

TYPESCRIPT TYPES
================

Sequent
  id: string
  antecedent: string[]
  consequent: string[]

ProofStep
  id: string
  rule: string
  sequent: Sequent
  description: string

Proof
  id: string
  title: string
  description: string
  steps: ProofStep[]
  createdAt: Date
  updatedAt: Date
  status: 'in-progress' | 'completed' | 'abandoned'

Project
  id: string
  title: string
  description: string
  proofs: Proof[]
  createdAt: Date
  updatedAt: Date

User
  id: string
  email: string
  name: string
  createdAt: Date

DEVELOPMENT TIPS
================

1. Path Aliases (tsconfig.json + vite.config.ts):
   $components/*  → src/components/*
   $pages/*       → src/pages/*
   $lib/*         → src/lib/*
   $types/*       → src/types/*

2. All pages accessible from Sidebar navigation

3. Components use Svelte's reactive declarations (<script>)

4. Tailwind utilities can be used directly in templates

5. TypeScript strict mode enabled for type safety

6. Responsive design with mobile-first approach

7. No backend logic - ready for API integration
